OVERVIEW                                                 *sidepanes-overview*

Sidepanes is a Neovim pane system for keeping a Markdown reference view,
Codex, Claude, and IPython in one reusable side window.

It is built around four pane types:

- Markdown viewer
- Codex terminal
- Claude terminal
- IPython terminal

The pane remembers scroll/cursor position in the Markdown viewer, reuses one
terminal session per tool, sends selected text into coding-agent prompt editors,
and can send lines or selections into IPython.

Most behavior is exposed through one facade:
>
    local sidepanes = require("sidepanes")
<

For the longer reference, see:
>
    doc/sidepanes.md
<

QUICK START                                           *sidepanes-quick-start*

Minimal setup:
>
    require("sidepanes").setup({
      commands = true,
      mappings = {
        global = {
          toggle = "<leader>pp",
          pick = "<leader>mP",
          headings = "<leader>fm",
          markdown = "<leader>p0",
          codex = "<leader>px",
          claude = "<leader>pc",
          ipython = "<leader>pi",
          focus = "<leader>pf",
          zoom = "<leader>pz",
          width_previous = "<leader>p-",
          width_next = "<leader>p+",
          width_picker = "<leader>pw",
          sticky_relative_width = "<leader>p%",
          send_ipython = "<leader>pl",
          ask = "<leader>pa",
        },
      },
    })
<

Open the switcher:
>
    :Sidepanes
<

Open help:
>
    :Sidepanes help
    :help sidepanes
<

COMMANDS                                                 *sidepanes-commands*

Default commands are registered when `commands = true`.

The main command is:
>
    :Sidepanes
<

With no arguments it opens the pane switcher. With arguments it dispatches
subcommands:
>
    :Sidepanes help
    :Sidepanes switch
    :Sidepanes toggle [file]
    :Sidepanes open {file}
    :Sidepanes markdown
    :Sidepanes pick
    :Sidepanes headings
    :Sidepanes codex [preset]
    :Sidepanes claude [preset]
    :Sidepanes tool {tool} [preset]
    :Sidepanes ipython
    :Sidepanes ipython-restart
    :Sidepanes ipython-clear
    :Sidepanes focus
    :Sidepanes zoom
    :Sidepanes width [value]
    :Sidepanes width next
    :Sidepanes width previous
    :Sidepanes width prev
    :Sidepanes width +
    :Sidepanes width -
    :Sidepanes width pick
    :Sidepanes width-pick
    :Sidepanes ask
    :Sidepanes ask-codex [preset]
    :Sidepanes ask-claude [preset]
<

Standalone commands:
>
    :SidepanesToggle [file]
    :SidepanesPick
    :SidepanesHeadings
    :SidepanesSwitch
    :SidepanesTool {tool} [preset]
    :SidepanesCodex [preset]
    :SidepanesClaude [preset]
    :SidepanesIPython
    :SidepanesIPythonRestart
    :SidepanesIPythonClear
    :SidepanesFocus
    :SidepanesZoom
    :SidepanesWidth [value]
    :SidepanesWidthPick
    :SidepanesAsk
    :SidepanesAskCodex [preset]
    :SidepanesAskClaude [preset]
<

Width commands accept columns, percentages, fractions, numeric ratios, deltas,
and aliases:
>
    :SidepanesWidth 100
    :SidepanesWidth 50%
    :SidepanesWidth 1/2
    :SidepanesWidth +10
    :SidepanesWidth -10
    :SidepanesWidth next
    :SidepanesWidth prev
    :SidepanesWidth +
    :SidepanesWidth -
    :SidepanesWidth pick
<

Ask commands are range-aware, so they can be used with visual selections or
explicit ranges:
>
    :'<,'>SidepanesAsk
    :'<,'>SidepanesAskCodex gpt55_high_fast
<

MAPPINGS                                                 *sidepanes-mappings*

Global mappings are optional and configured with `mappings.global`.

Example global mappings:
>
    <leader>pp       toggle pane
    <leader>mP       pick Markdown document
    <leader>fm       pick Markdown heading
    <leader>p0       show Markdown viewer
    <leader>px       show Codex
    <leader>pc       show Claude
    <leader>pi       show IPython
    <leader>pR       restart IPython
    <leader>pl       send line or visual selection to IPython
    <leader>pX       clear IPython
    <leader>pf       toggle focus between pane and previous window
    <leader>pz       toggle zoom
    <leader>p-       snap width down
    <leader>p+       snap width up
    <leader>pw       pick width
    <leader>p%       toggle sticky relative width
    <leader>ps       switch pane
    <leader>pa       ask picker from visual selection
<

Pane-local mappings are installed inside Sidepanes buffers:
>
    <space>0         show Markdown viewer
    <space>x         show Codex
    <space>c         show Claude
    <space>i         show IPython
    <leader>gg       toggle Markdown and last terminal, including terminal-input mode
    <C-g>            toggle Markdown and last terminal, including terminal-input mode
    <leader>gi       show IPython
    gf               smart go-to-file in the last non-pane window
    ll               send visual selection to IPython
    zz               toggle pane zoom
    aa               ask last coding agent from visual selection
    ax               ask Codex from visual selection
    ac               ask Claude from visual selection
<

Pane-local `gf` is implemented by `sidepanes.smart_gf`.

Markdown reflow is implemented by `sidepanes.markdown_reflow`. Sidepanes uses
that module path internally so the reflow code can be extracted later without
changing Sidepanes render logic.

CONFIGURATION                                       *sidepanes-configuration*

Sidepanes accepts grouped setup options and normalizes them to the internal
runtime config.
>
    require("sidepanes").setup({
      layout = {
        width = 100,
        zoom_text_width = 90,
        sticky_relative_width = false,
      },
      markdown = {
        wrap = false,
        auto_reload = true,
        reload_interval_ms = 1000,
        reload_badge_ms = 0,
        reload_badge = {
          text = "[RELOADED]",
          clear_on_interaction = true,
          hl = {
            fg = "CursorFG",
            bg = "WarningMsg",
            bold = true,
          },
        },
        sticky_heading = true,
        reflow = {
          enabled = true,
          cmd = { "mdfmt", "--stdin", "--width", "{width}", "--wrap", "always" },
          fallback = true,
          protect_tables = true,
          margin = 8,
        },
      },
      lifecycle = {
        focus_on_switch = true,
        focus_on_ask = true,
        shutdown_on_exit = true,
        shutdown_timeout_ms = 300,
      },
      terminal = {
        agent_resume_badge_ms = 0,
        agent_resume_badge = {
          text = "[RESUMED]",
          clear_on_interaction = true,
          hl = {
            fg = "CursorFG",
            bg = "DiagnosticInfo",
            bold = true,
          },
        },
      },
      validation = {
        enabled = true,
      },
      commands = true,
    })
<

To inspect the complete grouped defaults from Lua:
>
    vim.print(require("sidepanes.config").default_setup())
<

Legacy flat keys are still accepted. The grouped form is preferred for new
configuration.

When `markdown.auto_reload` is enabled, Sidepanes periodically compares a
content fingerprint of the active Markdown source file so same-size rewrites and
atomic saves are detected. `markdown.reload_interval_ms` controls the polling
interval. It also checks on focus and cursor-hold events, and when switching
back to the Markdown viewer. If the file changed on disk, the pane reloads it,
tries to return to the same or nearest matching line, and shows a `[RELOADED]`
badge in the winbar.

By default, the reload badge stays visible until key interaction inside the
Sidepanes Markdown pane. Set `markdown.reload_badge.clear_on_interaction` to
false to disable that behavior, or set `markdown.reload_badge_ms` to a positive
millisecond value to also hide it on a timer. `markdown.reload_badge.text`
changes the label. `markdown.reload_badge.hl.fg` and `.bg` accept hex colors or
Neovim highlight group names such as `CursorFG` and `WarningMsg`; group names
resolve to their configured colors for the generated `SidepanesReloaded`
highlight group.

Recovered Codex and Claude terminals show a `[RESUMED]` badge in the winbar by
default. Set `terminal.agent_resume_badge.clear_on_interaction` to false to keep
it visible, or set `terminal.agent_resume_badge_ms` to a positive millisecond
value to hide it on a timer. `terminal.agent_resume_badge.text` changes the
label. `terminal.agent_resume_badge.hl.fg` and `.bg` accept hex colors or
Neovim highlight group names for the generated `SidepanesResumed` highlight
group.

WIDTH                                                       *sidepanes-width*

The normal pane width can be configured or changed at runtime:
>
    require("sidepanes").set_width(100)
    require("sidepanes").set_width("50%")
    require("sidepanes").set_width("1/2")
    require("sidepanes").set_width(0.5)
    require("sidepanes").adjust_width(10)
    require("sidepanes").snap_width("next")
    require("sidepanes").width_picker()
<

`layout.width` accepts the same units. The default is 100 columns.

`layout.width_snap_points` controls `snap_width("next")` and
`snap_width("previous")`.

`layout.width_picker_points` controls the shorter picker shown by
`width_picker()`.

`layout.sticky_relative_width` keeps relative values such as "50%", "1/2", or
0.5 tied to the total Neovim width. When disabled, relative inputs are resolved
once to an absolute column width.

Width changes reflow Markdown only when the Markdown viewer is active. Terminal
panes resize without reflowing the hidden Markdown buffer.

PUBLIC API                                                 *sidepanes-api*

Stable public functions:
>
    setup(opts)
    get_config()
    open(path)
    toggle(path)
    close()
    is_open()
    focus_toggle()
    toggle_zoom()
    show_markdown()
    get_width()
    set_width(value)
    adjust_width(delta)
    snap_width(direction)
    width_picker()
    toggle_sticky_relative_width(enabled)
    text_width()
    toggle_wrap()
    pick()
    pick_headings()
    switch_picker()
    switch_to(target, opts)
    make_switch_entry(target, opts)
    open_terminal(tool_name, preset_name, opts)
    show_last_terminal(opts)
    toggle_markdown_terminal()
    open_ipython(opts)
    send_ipython(opts)
    clear_ipython(opts)
    restart_ipython(opts)
    ask(tool_name, preset_name, opts)
    ask_picker(opts)
    ask_last_coding_agent(opts)
    ask_current_coding_agent(tool_name, opts)
    shutdown_terminals(opts)
<

`switch_to()` is the stable switch API:
>
    require("sidepanes").switch_to("markdown")
    require("sidepanes").switch_to("codex")
    require("sidepanes").switch_to("claude")
    require("sidepanes").switch_to("ipython")
    require("sidepanes").switch_to("x")
    require("sidepanes").switch_to({ tool = "codex", preset = "gpt55_high_fast" })
<

Advanced helpers:
>
    show_last_terminal(opts)
    toggle_markdown_terminal()
    make_switch_entry(target, opts)
<

Compatibility aliases:
>
    show_last_agent(opts)
    toggle_markdown_agent()
<

Internal functions such as raw `switch(entry)` and `ask_with_entry(entry, opts)`
are intentionally not exposed on the public facade. Treat
`require("sidepanes.internal")` and `_state()` as private implementation hooks.

TOOLS                                                       *sidepanes-tools*

Built-in tools:

- Codex
- Claude
- IPython

Sidepanes keeps one terminal session per tool. Opening Codex with one preset
starts a new Codex terminal if needed. Later asks reuse that Codex session. If a
new preset is selected while the session is running, Sidepanes sends the tool's
configured model-switch command before sending the prompt.

Codex and Claude panes also track the agent process id and resumable session id
when the CLI exposes one. Opening Codex or Claude first reuses a live pane-owned
terminal buffer and Neovim job. If that live check fails because the terminal
exited, crashed, or its buffer was lost, Sidepanes builds a recovery candidate
from the current context, the remembered session record, or the latest matching
project transcript. Claude uses `~/.claude/sessions/<pid>.json` first and falls
back to the latest `~/.claude/projects/<project>/` transcript. Codex falls back
to the latest matching project transcript in `~/.codex/sessions/**`.

When a recovery candidate is launched, Claude receives
`claude --resume <session-id>` and Codex receives `codex resume <session-id>`.
Sidepanes checks whether any remembered previous PID is still alive, includes
the previous/new PID and session id in the notification when available, and
shows the recovered terminal badge. Sidepanes cannot reattach to a lost terminal
pty; recovery starts a new CLI process with the tool's resume command.

Each tool can define:
>
    label
    cmd
    args
    presets
    send_delay_ms
    switch_command
    exit_command
<

Default exit commands:
>
    Codex    /quit
    Claude   /exit
    IPython  quit()
<

ASK WORKFLOW                                             *sidepanes-ask*

Ask mappings and commands capture file name, project root, selected text, and a
snippet language. Markdown code fences are detected so a Python fenced block is
sent as Python rather than generic Markdown.

Asking opens a scratch prompt buffer. The prompt is sent only after the buffer
has been written and then quit. Quitting an unchanged prompt cancels and returns
to the previous pane/window state.

Useful calls:
>
    require("sidepanes").ask_picker({ visual = true })
    require("sidepanes").ask("codex", "gpt55_high_fast", { visual = true })
    require("sidepanes").ask_current_coding_agent("claude", { visual = true })
    require("sidepanes").ask_last_coding_agent({ visual = true })
<

MARKDOWN REFLOW                                 *sidepanes-markdown-reflow*

Markdown reflow is built in as:
>
    require("sidepanes.markdown_reflow")
<

Sidepanes uses that module internally for pane reflow, and users may configure
it for standalone buffer reflow:
>
    require("sidepanes.markdown_reflow").setup({
      external_reflow_cmd = { "mdfmt", "--stdin", "--width", "{width}", "--wrap", "always" },
      external_reflow_fallback = true,
      external_reflow_protect_tables = true,
      commands = true,
      mappings = {
        reflow = "<leader>mR",
      },
    })
<

Setup keys:
>
    external_reflow_cmd
    external_reflow_fallback
    external_reflow_protect_tables
    commands
    mappings.reflow
<

When `commands = true`, Markdown Reflow registers:
>
    :MarkdownReflow [width]
<

`external_reflow_cmd` may be a string, argv table, or function. The token
`{width}` is replaced with the computed target width.

If no external formatter is configured, internal paragraph reflow is used. If
an external formatter fails and `external_reflow_fallback` is true, Sidepanes
falls back to internal reflow. If fallback is false, the command reports an
error and leaves the buffer unchanged.

When `external_reflow_protect_tables` is true, Markdown table blocks are masked
before the external formatter runs and restored afterward.

HEALTH                                                     *sidepanes-health*

Run:
>
    :checkhealth sidepanes
<

The health check reports:

- loaded plugin state
- configured tools and presets
- external commands such as codex, claude, ipython, uv, and mdfmt
- optional dependencies used by enabled features
- command registration
- global and pane-local mappings, including expected modes
- malformed config values

Setup also performs lightweight validation by default. Disable it with:
>
    require("sidepanes").setup({
      validation = { enabled = false },
    })
<

DEPENDENCIES                                         *sidepanes-dependencies*

Sidepanes has no required Lua dependency for loading the core module. Feature
dependencies are optional and reported by setup validation and health checks.

Dependency contract:

- `telescope.nvim`
  - Required for: `:Sidepanes pick`, `:Sidepanes headings`,
    `:SidepanesPick`, `:SidepanesHeadings`, and their mappings.
  - If missing at runtime: the picker command reports a Sidepanes dependency
    warning and does not open the picker.
  - Setup validation: warns when the related command or mapping is enabled.
  - Health: warns that the optional Telescope module is missing.

- Treesitter Markdown parser
  - Required for: heading picker and Markdown code-fence context detection.
  - If missing at runtime: heading picker reports a dependency warning; ask
    context falls back where possible.
  - Setup validation: warns when heading picker command or mapping is enabled.
  - Health: warns when the parser cannot be created.

- `markview`
  - Used for: optional Markdown rendering decorations in the pane.
  - If missing at runtime: Markdown still opens; decorations are skipped.
  - Setup validation: no warning, because rendering is opportunistic.
  - Health: warns that the optional module is missing.

- `mdfmt`
  - Used for: external Markdown reflow only when configured through
    `markdown.reflow.cmd` or `external_reflow_cmd`.
  - If missing at runtime: Sidepanes falls back to internal reflow when
    fallback is enabled, otherwise reports an error.
  - Setup validation: no warning unless the command is part of tool config.
  - Health: ok/warn/error depending on command availability and fallback.

- `codex`, `claude`, `ipython`, and `uv`
  - Used for: terminal tools.
  - If missing at runtime: opening the related tool fails through terminal job
    startup.
  - Setup validation: warns when configured tool executables are missing.
  - Health: errors for missing configured tool commands; `uv` is informational
    because IPython may fall back to `ipython`.

Built-in modules:

- `sidepanes.smart_gf`
- `sidepanes.markdown_reflow`

Both are shipped inside `sidepanes.nvim` and are required health checks.

COMPATIBILITY                                       *sidepanes-compatibility*

Grouped setup keys are preferred for new configuration:
>
    layout
    markdown
    lifecycle
    validation
    commands
    mappings
    tools
<

Older flat setup keys remain supported by config normalization. They are kept
for compatibility, but docs and examples should use grouped keys.

Pane-local mapping keys `toggle_agent` and `toggle_agent_alt` remain supported
as compatibility aliases for `toggle_terminal` and `toggle_terminal_alt`. New
configuration should use the terminal-named keys.

Command aliases such as `:Sidepanes width prev`, `:Sidepanes width +`,
`:Sidepanes width -`, and the matching `:SidepanesWidth` aliases are supported
conveniences. They are documented and covered by tests, but the most readable
forms remain `next`, `previous`, and explicit width values.

Older advanced API names remain available as compatibility aliases:
>
    show_last_agent(opts)
    toggle_markdown_agent()
<

New code should prefer `show_last_terminal()` and `toggle_markdown_terminal()`.

==============================================================================
DESIGN NOTES                                             *sidepanes-design*

Sidepanes keeps mutable runtime state private. Feature modules receive explicit
dependency callbacks from the public facade; this keeps behavior testable while
avoiding global state leaks.

The public facade is intentionally smaller than the internal runtime. Prefer the
functions documented under |sidepanes-api|. Internal modules may change without
notice.
